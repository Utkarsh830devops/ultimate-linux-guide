# Networking Commands

1. **`ping google.com`** – Checks connectivity to a remote server.
2. **`ifconfig`** – Displays network interfaces (deprecated, use `ip`).
3. **`ip a`** – Shows IP addresses of network interfaces.
4. **`netstat -tulnp`** – Displays open/active network connections.
5. **`curl https://example.com`** – Fetches a webpage's content.
6. **`wget https://example.com/file.zip`** – Downloads a file from the internet.

4. **`Traceroute`** :- A network diagnostic tool that shows the path packets take from your computer to a destination host. - Use traceroute when you need detailed latency checks or are on Windows/macOS.

5. **`tracepath`** :-  It traces the path packets take to a destination host. - Use tracepath when you want a quick, root-free test or suspect MTU fragmentation issues (common in VPNs or tunnels).


6. **`mtr`** :- mtr (short for My Traceroute) is a powerful network diagnostic tool that combines the functionality of both traceroute and ping. It gives you a real-time, continuously updated view of the route packets take to a destination, along with latency and packet loss statistics at each hop.

7. **`nslookup`**  : nslookup (short for Name Server Lookup) is a command-line tool used to query DNS (Domain Name System) servers. It helps you find the IP address associated with a domain name, or the domain name associated with an IP address. It’s widely used for troubleshooting DNS issues.
